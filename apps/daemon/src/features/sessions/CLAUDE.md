# sessions

Mirror of the supervisor's session state, plus HTTP control surface. The `SessionRegistry` (scoped Effect layer in `sessions.repo.ts`) watches `<configDir>/daemon/roster.json` and one `<configDir>/jobs/<short>/state.json` per active session via `platform/fswatch.repo`; on each change it parses with `parseRoster` / `parseState` (pure decoders in `sessions.core.ts`, `@effect/schema` with `onExcessProperty: "ignore"`), reconciles the in-memory map, and publishes `roster.changed` / `session.created` / `session.state` / `session.removed` on `sseBus`. `readJsonWithRetry` handles non-atomic `state.json` writes (empty + `SyntaxError` retries × 5 × 50 ms). Routes: `GET /sessions`, `GET /sessions/:id`, `GET /sessions/:id/transcript` (reads `linkScanPath` JSONL, tail-truncated at 500 lines), `POST /sessions/:id/{stop,peek,rm,send}` — `send` accepts `{ keys }` (string, ≤4096 chars) and routes through `ShellRepo.send` which pty-attaches via the persistent attach pool.
