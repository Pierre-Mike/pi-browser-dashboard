You're looking at this project's own .pid/index.html, rendered as a sandboxed tab — the pid-apps feature dogfooding itself.

What just happened

This file lives at <repo>/.pid/index.html. When you open pi-browser-dashboard as a project in the dashboard, the daemon discovers it (zero config — presence of an index.html is enough), serves it under a strict Content-Security-Policy, and renders it here inside an <iframe sandbox="allow-scripts">: an opaque origin with no access to the parent page and no RPC bridge.

Drop your own

Anything you put under a project's .pid/ becomes a tab:

  • .pid/index.html → a default tab (this one)
  • .pid/<name>/index.html → a <name> tab
  • optional .pid/<name>/pid-app.json → { "title", "entry", "icon" }

Ship specs and plans as HTML, or run a small self-contained tool. Untrusted by design, so it's sandboxed and capability-free.

Full design + as-built record: specs/pid-html-extensions.html. Convention reference: the Per-project pid-apps section of AGENTS.md.
